AGENT 0 TEMPORARY HANDOFF: FORM GRAMMAR AND LOOKUP BUGS
========================================================

Date: 2026-07-01
Status: Read-only investigation findings. No implementation changes were made.


1. PRIMARY FINDING
==================

The current parser assumes that the first leading bold span is one complete
lookup expression.

That assumption is false for this dictionary.

One bold span may contain several different grammatical components:

* the lexical form;
* a Roman homograph number;
* the first Arabic sense number;
* a root/previous-form substitution operator (en dash or tilde);
* an additional derivative or compound form;
* parenthesized optional spelling material.

Therefore first_bold_text() must eventually be replaced by a small
dictionary-specific form grammar.


2. SCALE OF THE PROBLEM
=======================

A read-only scan of leading bold spans across the dictionary found:

    spans containing a Roman numeral:            10,622
    spans ending in an Arabic number:              3,815
    spans containing optional parentheses:         2,054
    Roman numeral followed by an Arabic number:      357
    Roman numeral followed by more bold words:       558
    bold spans containing an en dash:                271
    bold spans containing a tilde:                   172
    affix-acronym-affix hyphen patterns:               20

These categories overlap, but they demonstrate that this is not a small set of
manual exceptions.

4. REQUIRED FORM-EXPANSION STAGE
================================

Add a stage between semantic parsing and Yomitan term generation:

    PDF display form
        |
        v
    parse structural notation
        |
        v
    produce concrete lookup variants
        |
        v
    accent-fold lookup variants
        |
        v
    emit Yomitan rows/aliases

Suggested semantic fields:

    expression.pdf
    expression.components[]
    expression.lookup_variants[]
    expression.reading_variants[]
    homograph
    senses[]

For X(Y)Z, generate:

    XZ
    XYZ

For X(-Y)Z, generate:

    XZ
    X-YZ

Support multiple optional groups through a bounded Cartesian expansion.
Preserve the unexpanded PDF form for display and auditing.

Do not apply optional expansion to every parenthesis in an entry. Apply it only
inside a parsed bold lexical form. Parentheses in definitions and labels have
other meanings.


5. HOMOGRAPH AND SENSE PARSING
==============================

At a leading bold form:

1. Parse the lexical expression.
2. A whitespace-separated Roman numeral is a homograph marker.
3. A following whitespace-separated Arabic numeral is a sense marker.
4. Subsequent Arabic numerals introduce later senses.
5. En dash and tilde are substitution operators, not expression characters.
6. Additional bold lexical material after the Roman numeral may begin a form
   nested under that homograph.

Do not globally remove all Roman or Arabic numerals. Genuine expressions can
contain digits or apostrophe-year notation, for example A ’45. Parsing must use
position, whitespace, and dictionary context.

The renderer should emit the natural lexical expression while retaining
homograph and sense numbering in the glossary.


6. AFKIR AND NESTED YOMITAN LOOKUP
==================================

The afkir entry includes:

    détonator ~

The tilde expands to the nearest preceding bold form, afkiran. The phrase means
approximately:

    détonator afkiran

The visible word détonator should open the detonator entry in a nested Yomitan
popup.

Important finding:

The full PDF has a real bold headword on PDF page 261:

    détonator (D/E) detonator.

Therefore the full consecutive extraction should produce:

    primary expression: detonator
    reading:            détonator
    accented alias:     détonator

This specific nested lookup should work in the full dictionary once page 261
is included. It cannot work in the current pilot because the pilot does not
include page 261.

General rule:

* Preserve accents in definitions/examples.
* Accent-fold searchable forms.
* Emit accented aliases for accent-bearing PDF forms.
* After building the full form index, optionally audit accent-bearing tokens in
  definitions. If the accentless token resolves to a known form, verify that an
  accented alias exists or emit an internal query link.

8. OTHER HANDOFF WARNINGS
=========================

8.1 Full-corpus memory
----------------------

The current pilot script stores detailed line/span/character structures for all
selected pages in memory before completing the build. Running pages 21-1123
may consume excessive memory.

Before a production full run, stream pages or reduce each page to a lightweight
semantic line representation. Preserve parser state across page boundaries.


8.2 Overrides are not applied yet
---------------------------------

intermediate/overrides.csv is created and preserved, but the generator does not
currently read or apply it.


8.3 Cross-references can split across lines
-------------------------------------------

Example:

    MEMPERABU
    KAN

is currently recorded as two targets instead of MEMPERABUKAN. Cross-reference
resolution should join adjacent small-cap fragments across continuation lines
before lookup.

abandoned dashboard. It is not called and can be removed during cleanup.
