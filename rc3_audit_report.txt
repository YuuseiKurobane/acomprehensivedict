FOLLOW-UP CAUSAL AUDIT ADDENDUM — 2026-07-06
=============================================

QUESTION
--------

Does the typographic-run condition that broke "kemanisan" affect other
entries?

Answer: yes. "kemanisan" is one of 16 lexical lines missed by the same
line-start classification assumption. Four additional headwords are damaged
by the inverse version of the same one-character font-run problem.

No dictionary correction was made. This addendum records a read-only audit of
the existing PDF/debug evidence, human-readable intermediate, current rc3
archive, and initial rc3 Git artifact.

ROOT CAUSE
----------

extract_agent1.py classifies a line as a root or subentry start only when:

  1. its first meaningful PDF span is Bold or BoldItalic; and
  2. that span begins at a root or derivative horizontal anchor.

At page 635, the source line begins exactly at the derivative anchor, but its
first letter "k" is Indrev-Roman and only "emanisan" is Indrev-Bold. The line
therefore becomes a continuation of "(manis-)manisan". Layer 1 later preserves
the observed run split as Roman "k" plus Bold "emanisan", and Layer 2 emits the
broken visible text and no "kemanisan" lookup.

This is not limited to page or column boundaries. The same assumption fails
wherever a lexical start has an initial non-Bold span, regardless of its
position on the page.

FULL-CORPUS SCAN
----------------

The follow-up scan examined 157,313 unique source body lines in
intermediate\debug_full.json.

Exactly 17 lines classified as continuation begin at a configured root or
derivative anchor:

  - 16 are high-confidence lexical starts that were misclassified.
  - 1 is legitimate continuation prose in the "NW" entry:
      "For verbs beginning with ny-..."

The 16 lexical cases divide into:

  - 8 exact first-letter Roman + Bold-remainder splits, including "kemanisan".
  - 1 Roman punctuation + Bold-remainder split.
  - 7 lexical starts whose complete headword text is Roman.

An adjacent scan of successfully classified entry/subentry starts found four
more high-confidence one-character run errors in the opposite direction:
the line begins Bold, but its final headword letter is Roman. The parser emits
the Bold fragment as the lookup and moves the last letter into the definition.

No other high-confidence case of either geometry/style pattern was found.
Apparent mixed-style transitions with normal positive word spacing were
examples, labels, scientific names, or other legitimate definition content.

REVISED CONCLUSION AND RECOMMENDATION
-------------------------------------

RC3-AUD-001 should be broadened from one page-boundary defect to a systemic,
finite set of typographic-run classification defects:

  - 16 missed lexical starts caused by a non-Bold first meaningful span.
  - 4 truncated headwords caused by a final letter switching from Bold to
    Roman inside an otherwise valid start.

A repair should not special-case only "kemanisan". It should cover the 20
audited source lines and add regressions for every case. A blanket rule that
turns all Roman text at an anchor into an entry is unsafe because the "NW"
continuation line is a real counterexample. Geometry, lexical shape, adjacent
run continuity, alphabetical context, and exact reviewed exceptions should be
used together.

After correction:

  1. rebuild the full profile;
  2. run all pipeline tests;
  3. repeat the lookup and flattened-definition baseline diffs;
  4. confirm all corrected and accent-folded spellings are searchable;
  5. confirm the spellings remain visibly printed;
  6. inspect parent/child/related-form links for the affected families; and
  7. manually inspect the affected Yomitan entries, especially template
     expansion under duduk, jadwal, jotos, juang, layak, seru, skala, and
     timbal.

Finding RC3-AUD-002: source-contiguous runs acquire false spaces
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Root cause:

Layer 1 preserves font changes as separate marker records but does not
preserve whether two records were physically adjacent inside a source line.
Layer 2's _needs_space() then inserts a space between almost every pair of
rendered records. That is correct at ordinary semantic run boundaries, but
wrong when a word or number is split only because its PDF font changes.

Corpus method:

  - Scanned every source body line on pages 21–1123.
  - Selected adjacent, differently styled runs whose source text contains no
    boundary whitespace and whose bounding boxes touch or overlap (gap no
    greater than 0.8 point).
  - Excluded legitimate sense-number/definition and example/translation
    boundaries.
  - Confirmed the retained cases in the human intermediate and current
    flattened Yomitan definitions.

High-confidence impact:

  - 52 suffix boundaries are split: 51 English plural "-s" joins and the
    single derivational join "penghulu" + "ship".
  - 63 numeric joins on 59 source lines are split: 59 joins after a source
    decimal/thousands point and four digit-to-digit font changes.
  - Four other ordinary word joins are split.
  - Five mixed SmallCaps/Italic reference spellings are split.

Finding RC3-AUD-003: personal initials are converted into source-label badges
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Root cause:

Layer 1 calls tag_codes() for every complete Italic/BoldItalic span. A
one-letter initial or name fragment can therefore collide with a language
code in the tag table. _merge_punctuated_abbreviation_spans() handles only
the narrower case where every initial is already an isolated italic span; it
does not handle a first initial attached to the preceding example phrase.

Nine false tag conversions occur in eight source examples:

  PDF p135, pembélot:
    source "Kolonél F.D."
    current "Kolonél F. Dutch"

  PDF p151, besutan:
    source "sutradara N.A."
    current "sutradara N. Arabic"

  PDF p280, éks:
    source "K.M. Neptune"
    current "K. Minangkabau. Neptune"

  PDF p304, G II:
    source "Mt. G. Slamet"
    current "Mt. German. Slamet"

  PDF p404, incaran:
    source "U.S."
    current "U. Sundanese"

  PDF p452, mengabari:
    source "Pilot Lim C.L."
    current "Pilot Lim C. Latin"

  PDF p718, berpamit(an):
    source example variable "A."
    current "Arabic."

  PDF p901, bersaudara:
    source "P.S. Sumlang"
    current "P. Sundanese. Sumatra lang"
    ("S" and "Sum" are both converted to badges.)

The English translations beside these examples preserve the correct names or
initials and independently confirm that the badge readings are impossible.


Finding RC3-AUD-004: at least 107 mixed-style arrows lose cross-reference structure
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The human intermediate contains 122 literal "[Symbol] →" markers. A bounded
