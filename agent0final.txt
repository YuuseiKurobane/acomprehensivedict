1. GOAL AND CURRENT STATE
=========================

The goal is a high-quality Yomitan dictionary extracted from the PDF while
retaining enough semantic source data to repeatedly change parsing, colors,
and rendering without manually rebuilding term banks.

The current pilot:

* extracts PDF pages 21-30, 70, and 500;
* builds pretty semantic JSON chunks;
* builds a compact CSV form index;
* preserves a manual-override CSV;
* emits a format-3 Yomitan ZIP;
* emits a simple HTML word wall for scanning tests;
* preserves uncertain italics rather than guessing;
* expands source label abbreviations in displayed badges;
* creates accentless lookup forms while preserving PDF spellings/readings.

The current pilot is useful evidence, but its lexical-form grammar is not ready
for a full production extraction.

7. LOOKUP ACCENT NORMALIZATION
==============================

The PDF uses pronunciation marks such as é, while ordinary Indonesian text
usually uses unaccented spelling.

Lookup folding applies only to:

    headword.search
    form.expression.search
    Yomitan expressions
    internal query targets
    pilot_scan.html

Preserve accents in:

    *.pdf fields
    readings
    definitions
    examples
    scientific names
    foreign-language phrases
    source_markup

Example:

    PDF form:       abén
    search form:    aben
    reading:        abén

For accent-bearing forms, the pilot emits:

    expression=aben, reading=abén
    expression=abén, reading=""

Both rows share a sequence.

Implementation uses Unicode NFKD accent folding plus a small explicit
transliteration map for non-decomposing Latin letters.

9. CRITICAL PARSER DEFECT: LEADING BOLD SPAN IS NOT ONE EXPRESSION
==================================================================

Current code effectively assumes:

    first leading bold span == one lookup expression

This is false.

One leading bold span may contain:

* lexical form;
* Roman homograph number;
* first Arabic sense number;
* en-dash/tilde substitution operator;
* additional derivative/compound form;
* optional parenthesized spelling material.

Full-PDF read-only counts:

    spans containing Roman numeral:             10,622
    spans ending in Arabic number:                3,815
    spans containing optional parentheses:        2,054
    Roman numeral followed by Arabic number:        357
    Roman numeral followed by more bold words:      558
    leading bold spans containing en dash:           271
    leading bold spans containing tilde:              172
    affix-uppercase-acronym-affix patterns:            20

Categories overlap. This requires a grammar, not manual exceptions.


10. REQUIRED FORM GRAMMAR
=========================

Replace first_bold_text() with a parser that separates:

    lexical expression
    optional components
    homograph
    sense marker
    substitution operator
    additional inline forms

At a leading bold form:

1. Parse lexical material.
2. A whitespace-separated Roman numeral is normally a homograph marker.
3. A following whitespace-separated Arabic numeral is a sense marker.
4. Subsequent bold Arabic numerals introduce later senses.
5. En dash and tilde are operators, not expression characters.
6. Bold lexical material after a homograph marker may begin another form.

Do not globally delete numerals. Real expressions can contain digits or
apostrophe-year notation, such as A ’45.


11. OPTIONAL-FORM EXPANSION
===========================

Add an explicit stage:

    PDF form
      -> parse notation
      -> concrete variants
      -> accent-fold lookup variants
      -> Yomitan rows

For X(Y)Z:

    XZ
    XYZ

For X(-Y)Z:

    XZ
    X-YZ

Examples:

    mengabolisi(kan)
        -> mengabolisi
        -> mengabolisikan

    mengafkir(kan)
        -> mengafkir
        -> mengafkirkan

    absté(i)n
        PDF variants:    abstén, abstéin
        lookup variants: absten, abstein

    mengaduh(-aduh)
        -> mengaduh
        -> mengaduh-aduh

Support multiple optional groups with a bounded Cartesian expansion.
Only expand parentheses inside parsed lexical forms; parentheses elsewhere
have other meanings.

Suggested fields:

    expression.pdf
    expression.components[]
    expression.lookup_variants[]
    expression.reading_variants[]
