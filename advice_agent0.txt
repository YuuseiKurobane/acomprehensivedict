A COMPREHENSIVE INDONESIAN-ENGLISH DICTIONARY
IMPLEMENTATION ADVICE AND AGENT0 PILOT FINDINGS
================================================

Updated: 2026-06-30
Canonical source: acomprehensive.pdf
Pilot generator: pilot_extract_agent0.py
Semantic schema: agent0-semantic-pilot-2


1. CURRENT RECOMMENDATION
=========================

Use semantic, entry-centric JSON as the canonical intermediate format.

Do not use Yomitan term banks as master data, and do not use a per-character
PDF geometry dump as the normal working format.

Recommended pipeline:

    canonical PDF
        |
        v
    PyMuPDF layout-aware extraction and parsing
        |
        v
    pretty semantic JSON chunks
        |
        +--> compact CSV form index
        +--> manual override CSV
        +--> character audit
        +--> Yomitan renderer
        +--> other future outputs

This supports the expected iterative workflow:

* Change colors in acomprehensive_tags_map.csv and rebuild.
* Change Yomitan presentation code and rebuild.
* Improve a parsing rule and regenerate the semantic JSON from the PDF.
* Add durable manual corrections to overrides.csv.
* Never manually edit generated term_bank_*.json files.

ULTIMATE_fixed4.py remains the visual reference for badges, numbered senses,
internal links, example blocks, structured content, and bank-size handling.
The new intermediate data is not constrained by the KBBI CSV schema.

4. SEMANTIC INTERMEDIATE MODEL
==============================

The JSON is meant to be readable by humans and agents while retaining every
textual and structural fact needed to regenerate the dictionary.

It deliberately omits per-character boxes, font ascenders, and other PDF
debugging noise.

Each chunk has:

    schema_version
    group
    chunk_number
    entries[]

Each semantic entry contains:

Entry
    schema_version
    entry_id
    source_order
    entry_type
    source
        pdf_pages[]
        partial_start
        partial_end
    headword
        pdf
        search
        reading
        homograph
    labels[]
    forms[]
    sense_markers[]
    cross_references[]
    italic_annotations[]
    unassigned_content[]
    source_markup
    parser_warnings[]
    review_status
    manual_note

Form
    form_id
    parent_form_id
    form_order
    form_type
    expression
        pdf
        search
        reading
    homograph
    labels[]
    content[]
    source_markup

Content line
    line_id
    line_class
    text_pdf
    content[]

Ordered content node types currently include:

    text
        text_pdf
        style: roman | bold | italic | bold_italic
        optional text_extracted and extraction_repairs

    label
        code
        name
        color

    cross_reference_target
        display
        search
        style: small_caps

The content arrays preserve exact source order. This is more important than
forcing every item into a prematurely perfect sense model.

source_markup provides a compact readable rendering, for example:

    <b>abén</b> (<label code="Bal">Balinese</label>) cremation ceremony.

The markup is descriptive intermediate notation. It is not inserted into
Yomitan as unsafe HTML; the renderer builds schema-safe structured content.

6. CSV INDEX
============

intermediate/index.csv is the quick human work surface.

Columns:

    form_id
    entry_id
    parent_form_id
    form_type
    expression_pdf
    expression_search
    reading
    homograph
    labels
    pdf_pages
    chunk_file
    review_status

Example:

    expression_pdf     abén
    expression_search  aben
    reading            abén
    labels             Bal

Use the CSV for filtering, sorting, spreadsheets, quick audits, and locating
the corresponding semantic entry. Use the JSON chunk for nested content.

Trying to store all nested forms, senses, examples, and ordered styled runs in
one CSV would create repeated rows and fragile delimiter conventions. The
JSON-plus-index design avoids that while remaining accessible.

8. TWO DIFFERENT NORMALIZATION LAYERS
=====================================

Extraction repair and lookup normalization must remain separate.


8.1 Verified extraction repairs
--------------------------------

These repair incorrect legacy-font text extraction:

Raw       Count    Repaired
U+00A4    11,099   fi
U+00B6     4,141   fl
U+00AD    22,703   removed discretionary soft hyphen
U+00D8         2   1/5
U+2030         1   1/6
U+00B3         2   1/3
U+00BF         1   1/12

Examples:

    ¤rst       -> first
    dif¤culty  -> difficulty
    ¶ourishing -> flourishing
    in¶uence   -> influence

When a semantic text node contains one of these repairs, it retains:

    text_pdf
    text_extracted
    extraction_repairs[]

Thus the correction remains auditable without a separate geometry dump.

The fraction mappings were visually confirmed:

* PDF page 122 prints 1/4, 1/5, and 1/6.
* PDF page 456 prints 2 1/2 yards x 1 1/5.
* PDF pages 475 and 840 print 1 1/3.
* PDF page 334 prints 1/12.

No U+FFFE or U+FFFD replacement characters occur in PyMuPDF extraction.


8.2 Lookup normalization
-------------------------

The book uses é and related marks to indicate pronunciation distinctions, but
ordinary Indonesian text usually uses unaccented spelling. Yomitan must be able
to scan the ordinary form.

Lookup folding applies only to:

    headword.search
    form.expression.search
    Yomitan expressions
    internal-link targets
    pilot_scan.html

It does not alter:

    headword.pdf
    form.expression.pdf
    reading
    definitions
    examples
    scientific names
    foreign-language phrases
    source_markup

Examples:

    PDF/search/reading:
    abén  -> aben  / abén
    mengabén -> mengaben / mengabén
    à -> a / à

Implementation:

* Unicode NFKD decomposition;
* removal of combining marks;
* explicit transliteration for a small set of non-decomposing Latin letters;
* whitespace normalization;
* no global rewriting of definition/example text.


9. YOMITAN ACCENT BEHAVIOR
==========================

For a form whose PDF spelling differs from its natural lookup spelling, the
pilot emits two rows with the same sequence:

Primary natural-text row:

    expression = aben
    reading    = abén

Source-spelling alias:

    expression = abén
    reading    = empty

This allows both ordinary Indonesian text and dictionary-style accented text
to resolve.

The glossary continues to display the PDF spelling. Internal links target the
accentless lookup form.

The pilot increased from 850 to 957 term rows because accent-bearing forms now
receive aliases.


10. TAGS AND COLORS
===================

Tags and colors live in:

    acomprehensive_tags_map.csv

Columns:

    tag
    tag_rename
    type
    color

The Yomitan renderer displays tag_rename rather than the abbreviation:

    Bal -> Balinese
    coq -> Colloquial

The source abbreviation remains in badge hover metadata.

Multi-code spans such as "D/E leg" render as separate expanded badges.

The map is case-sensitive:

    Med = Medan
    med = medical
    Sg  = Singapore
    sg  = singular

Palette policy:

* part of speech follows the KBBI palette;
* colloquial/slang uses pink;
* obsolete/derogatory/vulgar uses red;
* acronym/abbreviation uses green;
* grammar uses purple;
* pronunciation/onomatopoeia/exclamation uses cyan;
* language, region, and domain labels use neutral gray;
* infrequent uses amber.

Native Yomitan tag banks do not contain color fields. Sense/form labels are
therefore styled structured-content spans, following ULTIMATE_fixed4.py.


11. RELATION TO BOOK CONVENTIONS (PAGES 15-17)
==============================================

"/" separates alternatives
    Preserve the punctuation and later model an AlternativeGroup.

"[X]" expands an acronym/initialism/abbreviation
    Attach an Expansion node to the preceding form.

En dash "–" refers to the headword/root
    Preserve it and later resolve it as RootSubstitution.

Tilde "~" refers to the nearest preceding boldface entry
    Preserve it and later resolve it to parent_form_id.

Boldface marks headwords and derivatives
    Becomes a Form with root/derivative type inferred from indentation.

Italic text can be an example, phrase, compound, abbreviation, or comment
    Always preserve italic style. Assign a semantic role only when confident.

Bold italic marks derivatives of compounds or negated words
    Preserve as bold_italic and later refine form_type.

"/.../" after a headword gives unpredictable pronunciation
    Later model as Pronunciation based on position and surrounding punctuation.

Roman numerals distinguish headwords with the same spelling
    Store as homograph, separate from expression.search.

Arrow plus small caps is a cross-reference
    Store display/search forms and later resolve against the complete form
    index. Render as a Yomitan internal query link.

Bold numerals identify meanings of the same word
    Currently retained as sense_markers; the next parser stage should form
    proper nested senses.

Lowercase a), b), etc. distinguish meanings of compounds
    Preserve now; later model as lettered subsenses.

Parentheses mean optional
    Later model as OptionalText for expressions. Parenthesized italic
    abbreviations near a headword are labels instead.

"(- X)" or "(~ X)"
    Later model the substitution operator and following material together.

"< >" surrounds an unpredictable root
    Preserve literally and later attach a root relationship.


12. ITALIC AMBIGUITY
====================

Pilot pages:

    21-30,70,500

Italic-run results:

    Total evaluated:                     1,602
    Confident labels:                      439
    Scientific-name candidates:             40
    Example-or-phrase candidates:           702
    Deliberately unclassified italic runs:  421

Unclassified does not mean lost. Those runs remain italic in semantic content
and in Yomitan.

Recommended policy:

1. Preserve every italic run.
2. Expand high-confidence abbreviation labels using the tag map.
3. Identify scientific binomials conservatively.
4. Pair examples only when punctuation, parser state, and following translation
   text support it.
5. Leave uncertain example/phrase material inline and italic.
6. Use an LLM only on a later audit queue if desired.

A faithful first dictionary does not require manual classification of every
italic phrase.

14. CURRENT LIMITATIONS
=======================

This remains a parser pilot.

Known limitations:

* Some headword lines contain alternatives joined by "and" which are not yet
  separately indexed.
* Sense markers are detected but definitions are not yet fully partitioned
  into nested Sense objects.
* Lettered compound subsenses are not yet modeled.
* Cross-reference targets split over visual lines can appear as separate
  unresolved targets, such as MEMPERABU + KAN.
* Example-versus-phrase classification remains conservative.
* Page selections without adjacent pages create partial records.
* Exact compound/negated-derivative subtypes need additional grammar.

The semantic content and source_markup retain the relevant evidence needed to
improve these later.
